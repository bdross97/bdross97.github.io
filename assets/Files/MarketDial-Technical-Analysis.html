Now that the data is centered and scaled, we can use the information gathered from the stores to cluster them in groups for sampling. There are numerous clustering algorithms and methods available, but for this analysis a Same Size K-means cluster method was selected. This allows for groups of equal sizes to be determined, ensuring we have a balanaced control set to pull from while also still detecting meaningful features that make up our groups to pull a treatment from.

We can initialize the clustering using this function:

kmvar <- function(mat, clsize=10, method=c('random','maxd', 'mind', 'elki')){
  k = ceiling(nrow(mat)/clsize)
  km.o = kmeans(mat, k)
  labs = rep(NA, nrow(mat))
  centd = lapply(1:k, function(kk){
    euc = t(mat)-km.o$centers[kk,]
    sqrt(apply(euc, 2, function(x) sum(x^2)))
  })
  centd = matrix(unlist(centd), ncol=k)
  clsizes = rep(0, k)
  if(method[1]=='random'){
    ptord = sample.int(nrow(mat))
  } else if(method[1]=='elki'){
    ptord = order(apply(centd, 1, min) - apply(centd, 1, max))
  } else if(method[1]=='maxd'){
    ptord = order(-apply(centd, 1, max))
  } else if(method[1]=='mind'){
    ptord = order(apply(centd, 1, min))
  } else {
    stop('unknown method')
  }
  for(ii in ptord){
    bestcl = which.max(centd[ii,])
    labs[ii] = bestcl
    clsizes[bestcl] = clsizes[bestcl] + 1
    if(clsizes[bestcl] >= clsize){
      centd[,bestcl] = NA
    }
  }
  return(labs)
}

Because of the nature of the selection method, we need to determine the right number of clusters for our algorithm based on the sample size. We can do this through a weighted mean computation for the relevant category revenue information. This will give us the required sample size to conduct a robust experiment.

stderrtest <- transactions %>%
  filter(date_week <= as.Date('2016-11-12', format = '%Y-%m-%d') & date_week >= as.Date('2016-07-17', format = '%Y-%m-%d') & product_id %in% prod_int$V1) %>%
  group_by(store_id) %>%

            prod_avg =  mean(revenue),

wghtmean <- weighted.mean(stderrtest$stdevtot, stderrtest$prod_avg)

sampsize <- function(stdev, marg, critval) { 
  return (((critval * stdev)/(marg))^2)
}

required_sample <- sampsize(wghtmean, .20, 0.841)
cluster_size <- round(nrow(pca_test)/ required_sample)
cluster_results <- kmvar(pca_test, clsize = cluster_size)
final <- as.data.frame(pca_test)

However, the average lift is still centered below 0, leading us to believe that this may not be the best choice for Client X to roll out in their stores when using the Pre-Test period as a baseline. To get a final conclusion we should look at the overall revenue lift from week-to-week over the test period.

As we can see from the plot above, using the pre-test period as a baseline to compare our treatment and control stores in the test period to presents some very skewed results. Using these inferences, we can safely say that using the pre-test period as a baseline is not the best idea in this case.

It should be noted that a baseline approach is not completely invalid, it may just have to be the same time periods for previous years. Using a different set of time for a baseline presents some bias in the results that would likely not be present if the same time period the test took place was used as a baseline from the year(s) previous.
